## Why

当前 HTTP 服务器默认绑定到所有网络接口 (0.0.0.0)，存在安全隐患，允许外部网络访问。为了增强安全性并满足用户自定义配置需求，需要将默认绑定限制在本机 (127.0.0.1)，同时提供系统配置界面让用户可以根据需要修改监听地址。

## What Changes

- **HTTP 服务器默认绑定地址修改为 127.0.0.1**，仅允许本地访问
- **BREAKING**: 默认行为改变 - 之前允许外部访问，现在默认只允许本地访问
- 新增系统配置页面，允许用户自定义：
  - 服务器监听地址 (bind address)
  - 端口 (如有需要)
- 配置持久化到配置文件
- 服务重启后配置生效

## Capabilities

### New Capabilities
- `configurable-server-host`: HTTP 服务器绑定地址配置能力，包括配置项定义、配置文件读写、配置验证机制
- `system-settings-page`: 系统配置页面 UI，提供用户界面用于修改服务器监听地址等系统级配置

### Modified Capabilities
- 无现有 spec 需要修改

## Impact

- **后端**: HTTP 服务器启动逻辑需要读取配置文件中的绑定地址
- **前端**: 新增【系统配置】页面路由和组件
- **配置文件**: 新增或修改配置文件格式以支持 bind_address 配置项
- **API**: 可能需要新增配置读写 API 端点
- **用户体验**: 首次启动后外部设备无法访问，需要用户在系统配置中修改地址
